# Eisenhower Matrix Advanced Task Classification Workflow

## Purpose
This document defines the strict workflow for classifying tasks in the Eisenhower Matrix using the user's Shared Story, long-term goals, and urgency. All teams and the AI assistant must use this as the reference for adding new tasks.

---

## Workflow Steps

1. **Reference the Shared Story**
   - Always use the user's Shared Story as the primary reference for determining what is important.
   - Never reinterpret or override the Shared Story.

2. **Check for Long-Term Goals**
   - Review the user's long-term goals (from their profile) to assess if the task supports these goals.

3. **Assess Urgency**
   - If the task has a deadline, use it to determine urgency.
   - If urgency is unclear, prompt the user for clarification (e.g., "Is this task urgent or can it wait?").

4. **Classify the Task**
   - **Important & Urgent**: Directly supports the Shared Story/goals and is time-sensitive.
   - **Important & Not Urgent**: Supports the Shared Story/goals but is not time-sensitive.
   - **Not Important & Urgent**: Does not support the Shared Story/goals but is time-sensitive.
   - **Not Important & Not Urgent**: Does not support the Shared Story/goals and is not time-sensitive.

5. **Clarification Workflow**
   - If the relevance to the Shared Story or urgency is unclear, always prompt the user for more information before classifying.

---

## Example Prompts
- "How does this task relate to your Shared Story or long-term goals?"
- "Is there a deadline for this task?"
- "Should this be prioritized now, or can it wait?"

---

## Enforcement
- All new tasks must be classified using this workflow.
- The AI assistant and all teams must refer to this document when adding or reviewing tasks in the Eisenhower Matrix.
